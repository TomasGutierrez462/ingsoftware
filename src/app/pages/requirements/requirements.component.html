Proyecto de Ingeniería de Software: Sistema de Reserva de Salas

Backlog del Producto

Visión General del Proyecto: Actualmente, el proceso de reserva de salas en el edificio de docentes es manual, utilizando papel y lápiz. Para optimizar este proceso, proponemos desarrollar un sistema de reservas que permita reservar salas de manera más rápida y desde cualquier ubicación. Es fundamental definir claramente los requisitos y funcionalidades para el sistema.

Requisitos del Proyecto:

  1. Desarrollo de la Base de Datos

    • Crear una tabla para registros de usuarios con campos: nombre, apellido, número de cédula y fecha de nacimiento.
    • Implementar soporte para varios formatos de número de cédula.
    • Desarrollar una tabla para gestionar los roles de los usuarios al registrarse.
    • Establecer una tabla para la información de las salas disponibles, incluyendo tamaño, número de sillas y ubicación.
    • Crear una tabla para las reservaciones con claves foráneas para el ID de usuario, ID de sala, fecha y hora de inicio y fin de la reserva.

  2. Desarrollo de la Página Web

    • Utilizar Node.js como entorno principal de ejecución.
    • Implementar el framework Angular para el desarrollo del frontend.
    • Utilizar TypeScript, CSS y HTML en el desarrollo.
    • Diseñar una interfaz web intuitiva para docentes de diferentes edades y personal del edificio.
    • Dividir el proyecto en frontend y backend para una gestión eficiente.
    • Asegurar una interacción fluida entre el usuario y el sistema de reservas.

  3. Optimización del Proceso de Reserva

    • Facilitar la reserva rápida desde cualquier ubicación.
    • Garantizar eficiencia en el proceso de reserva de salas.
    • Implementar notificaciones para confirmación de reservas.

  4. Seguridad y Privacidad

    • Integrar medidas de seguridad para proteger la información de los usuarios.
    • Implementar autenticación segura para acceder al sistema.

  5. Pruebas y Validación

    • Diseñar casos de prueba exhaustivos para cada funcionalidad.
    • Realizar pruebas de usabilidad con usuarios representativos.
    • Validar la funcionalidad del sistema en diferentes escenarios.
